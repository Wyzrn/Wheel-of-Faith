SvelteKit PageServerLoad pattern (RESEARCH.md Pattern 5):
```typescript
import { error } from '@sveltejs/kit'
import type { PageServerLoad } from './$types'

export const load: PageServerLoad = async ({ params, fetch }) => {
  const apiUrl = process.env.API_URL ?? 'http://localhost:3001'
  const res = await fetch(`${apiUrl}/api/characters/${params.id}`)
  if (res.status === 404) error(404, { message: 'This fate has been lost to the multiverse.' })
  if (!res.ok) error(500, { message: 'Failed to load character.' })
  return { character: await res.json() }
}
```
CRITICAL (RESEARCH.md Pitfall 1): The Vite proxy does NOT help SSR. The load function MUST hit `http://localhost:3001` directly via the API_URL env var, NOT `/api/...`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → SvelteKit (client fetch) | The Save & Share fetch body is fully client-supplied; server validates it in 04-01. No new trust boundary, only a new caller. |
| SvelteKit SSR → Fastify | Server-to-server fetch from `+page.server.ts`. Runs on the SvelteKit Node process, not in the browser — no CORS, no Vite proxy. `API_URL` is a server-only env var. |
| Browser → SvelteKit (share URL navigation) | A visitor with a share URL can render any saved character. This is the intended product behavior (BACK-02). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Information disclosure | `API_URL` accidentally exposed to client | mitigate | Use `API_URL` (no `VITE_` prefix) per planning_rules. `process.env.API_URL` is read in `+page.server.ts` only; SvelteKit will not bundle non-`VITE_`-prefixed envs into client code. `.env.example` documents this. |
| T-04-11 | Tampering | Stored XSS via character name on share page | mitigate | Svelte templates auto-escape `{character.name}` in `+page.svelte` and the existing `CharacterCard` template. No `{@html}` used anywhere. Server already enforces `maxLength: 40` (04-01 T-04-05). |
| T-04-12 | Denial of service | Save button spam from a single user | mitigate | Frontend disables button during in-flight fetch (`saving` state). Backend rate limit (04-01) is the authoritative protection. On 429, surface a clear message. |
| T-04-13 | Spoofing | Forged session_started_at to bypass 90s gate | accept | Matches 04-01 T-04-08. Frontend disables when local clock says &lt; 90s; backend re-validates. Documented anti-spam, not security (RESEARCH.md Pitfall 5). |
| T-04-14 | Repudiation | Save without clipboard support / private browsing | mitigate | URL pill text is fully selectable (`user-select: text`) so users can copy manually if `navigator.clipboard` throws. Wrap `writeText` in try/catch and silently fall back. |
| T-04-15 | Information disclosure | 500 errors leak stack traces on /character/[id] | mitigate | `error(500, { message: 'Failed to load character.' })` returns a plain message; SvelteKit hides internal errors in production builds. `+error.svelte` displays `$page.error?.message` only. |
</threat_model>
