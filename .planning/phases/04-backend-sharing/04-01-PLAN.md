<objective>
Stand up a standalone Fastify 5 + Mongoose 9 backend in a new `server/` directory that persists completed characters to MongoDB, returns nanoid(10) share IDs, enforces a 90-second minimum session length, applies per-IP rate limiting to POST /api/characters, and returns 404 for missing or soft-deleted characters.

Purpose: Phase 4 success criteria 1–4 all depend on this API existing and being correct under test. Wave 2 (frontend integration) cannot proceed without these endpoints.
Output: A self-contained `server/` package (ESM TypeScript, runnable via `tsx`) plus a passing backend test suite using `mongodb-memory-server` so no real MongoDB is needed in CI.
</objective>

ICharacter (Mongoose document):
```typescript
export interface ICharacter extends Document {
  shareId: string            // nanoid(10), unique, indexed
  name: string               // maxLength 40
  race: string               // indexed (gallery queries — Phase 5)
  archetype: string          // indexed
  overall_score: number      // 0–100, indexed (gallery sort)
  overall_tier: string       // one of the 28 grades
  spins: object              // Schema.Types.Mixed — full SpinResult[] JSON blob
  session_started_at: Date   // client-supplied ISO string, parsed to Date
  created_at: Date           // default Date.now
  deleted_at: Date | null    // default null, indexed
  share_in_gallery: boolean  // default false
}
```

POST /api/characters request body (Fastify JSON schema):
```typescript
{
  name: string                  // required, maxLength 40
  race: string                  // required
  archetype: string             // required
  overall_score: number         // required, 0–100
  overall_tier: string          // required
  spins: unknown[]              // required, array (heterogeneous SpinResult shape — store as Mixed)
  session_started_at: string    // required, format: 'date-time'
  share_in_gallery?: boolean    // optional, default false
}
```

POST /api/characters responses:
- 201 → `{ shareId: string, url: string }` where url is `/character/${shareId}`
- 422 → `{ error: 'Session too short', message: string }` when (Date.now() - session_started_at) / 1000 < 90
- 429 → `{ statusCode: 429, error: 'Too Many Requests', message: string }` from @fastify/rate-limit after > 5 requests / 10 min / IP
- 400 → Fastify default schema validation error for malformed body

GET /api/characters/:shareId responses:
- 200 → lean Character document (object)
- 404 → `{ error: 'Character not found' }` when no doc OR soft-deleted (pre-find hook handles the latter)

  // ...
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Fastify | Untrusted JSON body crosses here (POST /api/characters). All fields are client-supplied including session_started_at. |
| Reverse proxy → Fastify | In dev, Vite's proxy hop; in prod, an LB hop. The real client IP arrives in X-Forwarded-For; rate limit decisions depend on reading it correctly. |
| Fastify → MongoDB | Driver boundary. `spins` is stored as Mixed — a malicious client could place `$`-prefixed keys inside spin objects. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | Rate limiter IP detection | mitigate | `trustProxy: true` on Fastify so `request.ip` reads X-Forwarded-For correctly; documented as a v1 acceptance for local dev (no public CIDR allow-list yet). Without this, all users share 127.0.0.1 and one user spams everyone offline (RESEARCH.md Pitfall 2). |
| T-04-02 | Tampering | POST body validation | mitigate | Fastify Ajv JSON schema on POST /api/characters: required fields, `maxLength: 40` on name, numeric bounds on overall_score (0–100), `format: 'date-time'` on session_started_at. Rejects malformed bodies with 400 before any DB write. |
| T-04-03 | Tampering | `spins` Mixed-field NoSQL operator injection | mitigate | The route reads `body.spins` and stores it directly into a Mixed field. Mongoose does NOT execute operators on insert — only on query. The current API never queries by spin content, so the risk is bounded. Re-evaluate if Phase 5 gallery adds spin queries. (RESEARCH.md Security Domain row 1.) |
| T-04-04 | Repudiation | Anonymous saves | accept | No auth in v1 per REQUIREMENTS.md "No login / accounts". Share link IS the identity. Rate limit + 90s guard cap spam. |
| T-04-05 | Information disclosure | Stored XSS via character name | mitigate | `maxLength: 40` on name. SvelteKit auto-escapes text in templates (Wave 2 renders this), and CharacterCard already does. No raw HTML rendering of name. |
| T-04-06 | Denial of service | Excessively large POST body | accept | Fastify default body limit is 1 MB; a 35-spin payload is ~10 KB. No change needed (RESEARCH.md Security Domain row 3). |
| T-04-07 | Denial of service | Save-button spam from one IP | mitigate | `@fastify/rate-limit` per-route: max 5 in 10 min. Returns 429 with retry-after header. |
| T-04-08 | Denial of service | Bypass 90s session guard with replayed timestamp | accept | session_started_at is client-supplied. Rate limit (5/10min) provides real protection (RESEARCH.md Pitfall 5). Acceptable for v1 anti-spam (not a security control). |
| T-04-09 | Elevation of privilege | Access to soft-deleted characters via GET | mitigate | Mongoose pre(/^find/) hook unconditionally filters `deleted_at: null`. GET /api/characters/:shareId for a deleted doc returns 404 (BACK-03 verbatim). |
</threat_model>
