<objective>
Wire the Story Mode spin loop end-to-end. The user clicks "Enter Story Mode", spins through the queue, and the resulting character is auto-named, added to the roster (cap-enforced), and shown in the roster view. State during the spin lives under a NEW localStorage key `story_session` — completely isolated from `wof_session`.

Purpose: This is the integration that completes Phase 7's success criterion #2 (success criterion: "Spinning a new character inside Story Mode runs a full constrained wheel session and on completion lands in Story Mode — the main game is never touched") and #3 ("The resulting character is auto-named, added to the Story Mode roster, and persists across page reloads"). Plans 01 and 02 built the data layer and shell; this plan closes the loop.

Scope decisions (called out explicitly to bound the work):

- The Story Mode spin queue uses `buildInitialQueue()` from `$lib/game/spinQueue` AS-IS. Dynamic queue mutations specific to the main game (race subType/class branching, archetype custom-ability pool branching, redemption mutations, weakness count derivation, weapon-enchantment splicing, etc.) are NOT replicated in Phase 7. Phase 7's Story Mode spin loop runs the base queue and the segment resolver uses `getSegmentsForCategory(def.category)` directly. This is acceptable because Phase 8 will add tier gating and the per-race custom pools are a refinement, not a Phase 7 success criterion. If a race has an explicit `abilities` pool on its content entry, use it; otherwise fall back to the category default. Document any other simplifications in the SUMMARY.
- No Redemption Spin in Phase 7's Story Mode (defer to Phase 8+). The queue's `redemption` step is skipped (treated as a no-op — the spin definition for redemption is filtered out before the queue runs).
- Mid-session resume: if `story_session` localStorage holds an in-progress session at mount, offer a Resume prompt similar to the main game's. If the user declines, clear the key and start fresh.

Output:
- `src/lib/story/storySession.ts` — story session shape + CRUD + `buildRosterEntryFromResults()`
- `src/lib/story/storySession.test.ts` — unit tests for CRUD round-trip and entry building
- `src/components/story/StorySpinView.svelte` — the spin loop component (SpinWheel + queue orchestration + completion callback)
- Updated `src/routes/story/+page.svelte` — mounts StorySpinView when `view === 'spin'`, replaces the Plan 02 stub
</objective>
