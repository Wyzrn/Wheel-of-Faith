<weakness_logic>
## Weakness Sentinel Logic (RESEARCH.md Pattern 8)

When the spin with category === 'weakness' lands, it is the SENTINEL — it does NOT record
a weakness result. Instead it:
1. Rolls a weakness count using the race's weaknessProbabilityModifier
2. Splices that many actual 'weakness' spin definitions immediately after the sentinel
3. Records no result itself (or records a meta-result with resultLabel = 'Weakness Sentinel')

Baseline probabilities for count: [0: 10%, 1: 45%, 2: 35%, 3: 10%]
Modifier effect: multiply each non-zero probability by the modifier, renormalize.
High modifier (1.5): shifts weight toward 2–3 weaknesses.
Low modifier (0.5): shifts weight toward 0–1 weaknesses.

Implementation: use weightedRandom() on an array of { label: '0', weight: }, { label: '1', ... }
entries where weights are derived from the baseline probabilities scaled by modifier.

The actual weakness spins that are spliced in have category 'weakness' (not 'weaknessSentinel')
so they present as normal weakness spins in the queue. The sentinel itself is consumed silently.
</weakness_logic>
