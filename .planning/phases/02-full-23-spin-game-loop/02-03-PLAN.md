<label_authoring_rules>
## FlavorLabel Authoring Rules (CORE-04 + RESEARCH.md Pitfall 5)

Every stat file MUST follow these rules:
1. Exactly 20 unique labels per tier grade (28 tiers × 20 labels = 560 entries per stat file)
2. weight: 1 for every entry — equal weight across all tiers (do not down-weight F- or God)
3. score: use the canonical midpoint for each tier (see table above)
4. color: always set to the tier's CSS variable string (see mapping above)
5. tone: F- and F tier labels are weak/embarrassing/absurd; God tier labels are transcendent/cosmic
6. No duplicate labels within a stat file; avoid same label across different stat files where possible
7. Labels reflect the STAT being measured — Strength F- is weak, Strength God is godlike

## Per-Stat Flavor Tone Guide

**Strength:** F- = "Wet Noodle Arm", "Cannot Open Pickle Jar", "Struggles with Gravity"; God = "Continental Divide", "Universe-Shattering Punch", "Physics? What Physics?"
